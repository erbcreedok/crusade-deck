# Worklog: режим раздачи, анимации, готовность, рассадка

Сводка работ по клиенту/серверу вокруг режима раздачи (deal mode), переносов карт,
кнопки «Готов», автораздачи и стабильной посадки за столом. TDD: новая логика
начиналась с падающих тестов (`*.test.ts` рядом с модулями).

---

## 1. Анимация переноса карт по столу

**Проблема.** При раздаче / сборе / сбросе карты телепортировались на место
назначения.

**Решение.** Призраки-спрайты летят дугой; схема уже обновлена, визуал догоняет.

| Событие | Источник | Поведение |
|--------|----------|-----------|
| Раздача DnD | дилер отпускает карту | полёт с пальца → рука/место |
| `card_moved` | сервер после `deal_card` | полёт у всех; дилерский дроп не дублируется |
| `hands_collected` | сбор рук | облёт с мест в колоду + `counts` |
| `deck_reset` | сброс колоды | тот же облёт, что у сбора |

**Клиент**
- `client/src/game/cardFlight.ts` — математика дуги (`cardFlightPose`)
- `anim.cardMove` в `anim/config.ts` — длительность, подъём, stagger
- `RoomEngine`: очередь `cardFlights`, `playDealFlight` / `playCardMoved` /
  `playCollectAnim`, тик `stepCardFlights`, keep-awake пока есть полёты
- своя рука: карта скрыта до приземления (`flightCards` + `reconcileHandByIdentity`)
- чужие места: `seatFlightBias` — стопка/счётчик не прыгают вперёд полёта
- `RoomScreen` / `RoomCanvas` — сигналы `cardMovedSignal`, `collectSignal` (+ counts)

**Сервер**
- `deal_card` → `broadcast("card_moved", { moves })`
- `collect_hands` / `reset_deck` → `order` + `counts` для анимации

## 2. «Готов» включает/выключает дроп-зону

**Правило.** В режиме раздачи карта принимается только если цель **готова**
(или это дилер — см. §3).

**Клиент**
- `pickDealTarget(..., readyIds)` — неготовый seat не accept
- ховер по неготовому всё равно ловится → оверлей «Неа»
- дроп в неготового → отбой `startCardReject` + текст **«нииизя»**
- автораздача пропускает снявших «Готов»

**Сервер**
- `deal_card` отклоняет цель без `isReady` (кроме дилера как цели)

**UI ховера**
- готов → подпись «раздать», плотный цветной оверлей поверх бокса
- не готов → «Неа», тот же плотный оверлей, дроп запрещён

Модуль цветов/подписей: `client/src/game/dealReadyTint.ts`.

---

## 3. Дилер всегда готов

- при назначении дилера: `isReady = true`
- сообщение `ready` от дилера игнорируется (нельзя «снять» готовность)
- новый дилер после голосования тоже сразу готов
- `isDealReady(isReady, isDealer)` — единая проверка на клиенте
- автораздача и `deal_card` учитывают дилера как всегда принимающего

---

## 4. Цвет зон готовности (стиль, не контент)

| Состояние | Цвет рамки зоны |
|-----------|-----------------|
| Готов | жёлтый (`#d9b154`) |
| Не готов | серый (`#8a9490`) |

**Idle:** только тихая обводка, без заливки и без тонирования карт/счётчиков.  
**Hover при раздаче:** плотный оверлей цвета зоны + текст действия.

**Багфикс:** своё место вырезано из `seats` → готовность руки бралась из пустоты
и всегда была «серой». Исправлено: `setSelfDealState(selfReady, selfIsDealer)`
из `RoomScreen` → `RoomCanvas` → `RoomEngine`.

## 5. Автораздача

**Было:** два круга по одной карте, интервал ~667 мс.  
**Стало:**

- **2 карты в секунду** (`AUTO_DEAL_INTERVAL_MS = 500`)
- старт с соседа слева от дилера, дальше по кругу
- каждому **по две**, потом следующему (`autoDealPlan`)
- пока колода не пуста или нажат **STOP**
- только готовые (+ дилер); порядок из серверного `seatOrder`

Файлы: `client/src/game/dealing.ts`, драйвер в `RoomScreen.tsx`.

## 6. Стабильная рассадка (seatOrder)

**Проблема.** Боты тестовой комнаты попадали в `Map` раньше дилера. Каждый клиент
фильтровал «себя» из списка → второй человек оказывался в одном слоте у обоих
(«я слева от тебя» с обеих сторон).

**Решение — серверный круг `GameState.seatOrder`:**
- дилер в **голове** круга
- боты следом
- остальные join’ы в хвост
- takeover: замена sessionId в круге; kick: удаление

**Клиент**
- `seatsForViewer(tableOrder, selfId)` — чужие места по часовой, начиная с
  соседа слева от меня
- посадка «П» (`layoutSeats`) получает уже этот порядок
- автораздача / анимации сбора тоже опираются на `seatOrder`

Файлы: `server/.../GameState.ts`, `CardRoom.ts`, `TestRoom.ts`,
`client/src/game/seatOrder.ts`, проводка в `RoomScreen.tsx`.

**Важно:** старые комнаты без `seatOrder` нужно пересоздать.

## 8. Сопутствующее по режиму раздачи (в том же объёме диффа)

Кратко то, что уже легло в код рядом с пунктами выше (веер колоды/руки, чужие руки,
роли панели и т.д.):

- две стопки в движке: колода (центр) + своя рука (`setHand`)
- чужие руки на местах: стопка / веер, `handOpen` vs `handFanned`
- веер колоды на столе (`deckFanned`, дилер); не-дилер — peek / «низяяя»
- `barActions`: дилер — тасовка / автораздача; игрок — Готов / Ждите
- сортировка руки (`sortHand`), запрет открыть колоду не-дилеру (`forbidDeckOpen`)
- геометрия веера/кнопки collapse, fan angle для малого числа карт

## Как проверять

1. Тестовая комната (боты) + второй клиент по коду — рассадка согласована:
   слева от дилера первый бот; у второго игрока слева дилер.
2. «Готов» → жёлтая рамка; «Не готов» → серая; ховер с оверлеем; дроп в серого → «нииизя».
3. Дилер всегда жёлтый / всегда принимает карты.
4. DnD раздача и автораздача — карты летят; авто: по две, 2/с, по кругу до конца/STOP.
5. Собрать карты / сбросить колоду — облёт в центр.
6. Код комнаты — 4 цифры.
