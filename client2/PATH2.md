# Путь 2 — единый пул карт стола

Цель: убрать «призрачные» перелёты между боксами. Сейчас движок держит **четыре**
независимых `CardPile` (колода / рука / сброс / зона), каждый владеет своими спрайтами и
реконсилится сам по себе. Переход карты из бокса в бокс = уничтожить спрайт в одном +
создать в другом + временный спрайт-призрак, летящий между ЦЕНТРАМИ боксов
(`cardMoveAnchor`). Отсюда «летит в центр», рассинхроны и лишняя машинерия.

Путь 2: **один спрайт на идентичность карты на весь стол.** Боксы — это лишь якоря покоя.
Переход между боксами = `body.setTarget(новый якорь)` на ТОМ ЖЕ спрайте → настоящий
бесшовный перелёт колода↔зона↔рука↔сброс↔веер, без призраков и пересоздания.

## Граница, которая остаётся

Чужие руки/места — агрегат (счётчик), спрайтов чужих скрытых карт нет. Значит пул — это
все карты, которые ЭТОТ клиент видит поодиночке: колода, сброс, зона, вееры, своя рука, +
карты в перелёте к/от мест. На границе видимости карта **появляется** (спавн у якоря
источника → полёт к слоту) или **уходит** (полёт к месту → ретайр в счётчик). Это
единственное место, где остаётся «спавн/ретайр».

## Ядро — `TablePool` (engine/tablePool.ts)

Обобщение `CardPile` с одного пула на весь стол. Держит `Map<cardId, CardVisual>` и плоский
список слотов `{card, box, within}`. `apply(next)`:

- карта уже в пуле → оставляем спрайт, `setTarget(anchor(slot))` — летит из старого места
  (любого бокса) в новое; ЭТО и есть бесшовный перелёт;
- новой карты в пуле нет → создаём, ставим у `spawnAnchor(slot)` (якорь источника) и
  `setTarget(anchor(slot))`;
- карта пропала из `next` → `onLeave` (ретайр / анимация ухода).

`anchor(slot, ctx)` даёт движок (знает живую раскладку: deckStack / handRow / discardHeap /
playGrid). Пул — чистый (без Pixi), тестируется на фейковом `CardVisual`.

Слот несёт `box` строкой (`"deck"|"hand"|"discard"|"play:N"`) и `within` — индекс внутри
бокса. Порядок внутри бокса задаёт `within`; порядок боксов — сам `box`.

## План интеграции в RoomEngine (пошагово, держим тесты зелёными)

1. **[сделано]** `TablePool` + юнит-тесты (this commit).
2. `assembleTable(deck, hand, discard, play)` — из четырёх массивов состояния собрать единый
   список слотов (playFlat уже даёт плоский порядок зоны). Чистая функция + тест.
3. `applyTable()` в движке: один reconcile пула на все боксы. `anchor(slot)` диспетчеризует
   в существующие `restTarget/handRestTarget/discardRestTarget/playRestTarget`.
4. Перевести `setDeck/setHand/setDiscard/setPlay` на буферизацию + один `applyTable` (боксы
   приходят порознь через useEngineEffect — нужен единый прогон, иначе идентичность рвётся
   на границе, как сейчас).
5. Удалить призрачную подсистему для ВИДИМЫХ переходов (`enqueueCardFlight` и производные
   остаются только для границы видимости — спавн/ретайр у мест).
6. Сохранить спецанимации поверх пула: тасовка, скрамбл, переворот (flip вдоль перелёта уже
   есть в client), `collect_hands`, free-`take`; защиты `deckRev`/`set_hand_order`/драг.
7. Переворот на границе бокса: если сторона карты в новом боксе отличается (закрытая колода
   → открытая рука; лицо → рубашка сброса) — перелёт несёт flip (логика startFace≠faceUp
   уже перенесена из client).

## Риски

- Порядок применения: сейчас боксы независимы; единый reconcile должен сохранить
  оптимистичный драг и игнор устаревших эхо (revisions).
- Спецанимации завязаны на пуловую собственность — переносить аккуратно, по одной.
- Тестируемость: реальный Pixi в jsdom не идёт; ядро (пул, сборка слотов, якоря) — чистое и
  тестируется, визуал проверяется на устройстве.
