# Режим раздачи: роли, рогатка, веер на столе, автораздача

## Context

Сейчас в комнате нет понятия «кто что делает». Колода лежит в центре, дилер может утащить
её в руку, раздача — одна кнопка «Раздать», которая мгновенно делит колоду поровну
(`start_game` → `dealCards`). Панель действий (`barActions.ts`) знает только про выделенную
колоду и предлагает «В руку» / «В центр».

Нужен **режим раздачи** — состояние комнаты, включённое по умолчанию, в котором никто не
видит номиналов, пока карта не попала ему в руку; дилер раздаёт руками (рогаткой из
закрытой колоды или драгом из открытого веера) либо автоматически; у обычных игроков UI
сведён к «готов / не готов». Вход и выход из режима — следующая задача, здесь режим просто
включён по умолчанию.

## Пробел, который вскрывается этой задачей

**Розданные карты сейчас негде показать.** Движок рисует ровно одну коллекцию — `deck`
(`RoomEngine.setDeck`, `RoomCanvas:152`). То, что в UI называется «рукой», — это та же
колода, притащенная в зону руки (`deckZone === "hand"`). А `Player.hand`, куда падают
розданные карты, не рисуется вообще: виден только счётчик на месте игрока
(`RoomScreen:96 handCount`).

Раздавать в невидимую руку бессмысленно, поэтому в план входит разделение на **две стопки**:
колода в центре и МОЯ рука внизу. Требование «дилер видит только один веер одновременно —
либо колода, либо рука» это подтверждает: вееров теперь два.

## Принятые решения

1. Веер колоды на столе — **рубашкой ко всем, включая дилера**: он для слепой тасовки.
2. Промах рогатки — карта трясётся у края экрана, надпись **«Мииииимоо»**, возврат в колоду.
3. `start_game`/`dealCards` — **убрать из UI, механику оставить**.
4. В режиме раздачи колоду **нельзя взять в руку**, драг для переноса выключен целиком:
   единственный драг на закрытой колоде — натяжение рогатки.
5. Автораздача — по кругу **от соседа, дилер последний**, двумя кругами по одной карте.
6. Веер на столе раскрывается **двойным тапом**, сворачивается только стрелкой, и стрелка
   именно **сворачивает веер в колоду** (в руке она снимает фокус — смыслы разные).
7. **Один веер за раз**: раскрытие колоды сворачивает руку и наоборот.
8. **Рогатка требует хорошего натяга**: ниже порога выстрела нет вообще; натянул слабо для
   своего экрана — карта не долетает, тормозит на полпути и возвращается с надписью.
9. **Сбор карт** — дилерское действие: у всех играет анимация, карты летят в колоду по
   часовой стрелке от дилера.
10. **Сортировка руки** (по масти / по номиналу) — любому игроку при раскрытом веере руки;
    считается на клиенте, на сервер уходит готовый порядок.

## Что переиспользуется

- `game/fan.ts` — вся геометрия веера; сейчас привязана к зоне руки только через
  `RoomEngine.fanGeom()`, её нужно параметризовать зоной.
- `game/collapseButton.ts` — вписывание кнопки в карман под веером (годится обоим веерам).
- `game/swipeShuffle.ts` — `swipeVelocity`, `isSwipeDown`, свайп-тасовка.
- `game/flip.ts` `stretchOffset` — резина для натяжения рогатки.
- `RoomEngine.splashAnim` / `startShuffleAnim` — шаблон полёта карт по дуге.
- `RoomEngine.showNotice()` — оверлей для «Мииииимоо» и «не долетело».
- `dropZones.pickSeat()` + `seatLayout.ts` — прямоугольники мест: по ним целится рогатка
  и к ним же летят карты при сборе.
- `game/handView.ts` — «свою руку владелец видит всегда», поэтому «получил карту → сразу
  видишь номинал» отдельного кода не требует.
- `deckOrder.isPermutationOf` (сервер) — валидация присланного порядка руки.
- `game/card.ts` `parseCard`/`SUITS` — основа сортировок.
- `collect_hands` (сервер, уже есть) — механика сбора; добавляется только анимация.

## План

### Этап 1. Две стопки в движке (фундамент)

`RoomEngine` обобщается с одной коллекции карт до двух: `deck` (центр) и `hand` (моя рука).
Всё, что сейчас завязано на `this.cards`, получает владельца-стопку: раскладка
(`restTarget`/`fanTarget`/`rowTarget`), веер, драг карты, тасовка, перевороты.
`RoomCanvas` получает вторую пару входов (`hand`, `handFanned`), `RoomScreen` берёт
`Player.hand` из состояния.

Правило одного веера: раскрытие одного веера сворачивает второй (и локально, и в
сообщении на сервер для колоды).

### Этап 2. Серверная модель режима

`server/src/GameState.ts`: `dealMode: boolean = true` (инверсия нынешнего `revealMode`),
`deckFanned: boolean = false` — веер на столе, видимый всем.

`server/src/CardRoom.ts`:
- `set_deck_fanned { open }` — дилер, колода в центре;
- `deal_card { card, to, rev }` — дилер; карта из колоды уходит в руку игрока;
- `set_hand_order { order }` — свой порядок руки, принимается как перестановка своей руки;
- `move_deck` запрещён в режиме раздачи;
- `collect_hands` — уже есть, добавляется рассылка fx-события сбора;
- `start_game` остаётся, из UI убирается.

Чистая логика в `server/src/handRules.ts`: `dealCardTo(deck, hands, card, to)`.
Тесты: уходит ровно одна карта, порядок остальных цел, чужая/несуществующая отбивается,
не-дилер игнорируется, `move_deck` в режиме раздачи не двигает колоду.

### Этап 3. Веер на столе

`fanGeom()` параметризуется зоной (`handZone` | `centerZone`); `fanned()` учитывает обе
стопки. Двойной тап по колоде в центре → `set_deck_fanned {open:true}`; стрелка под веером
на столе → `{open:false}`. Карты веера на столе всегда рубашкой. Свайп-тасовка разрешена
**только** в режиме раздачи.

### Этап 4. Рогатка

Новый чистый `client/src/game/sling.ts`:
- `slingPull(start, current, maxPull)` — вектор натяга с ограничением;
- `slingPower(pullLen, maxPull)` → 0..1, и `MIN_LAUNCH_POWER` — ниже него выстрела нет;
- `shotAngle(pull)` — угол, противоположный натягу;
- `pickShotTarget(angle, seats, from, toleranceRad)` → id или `null`;
- `shotReach(power, screenDiag)` → дальность в пикселях: хватает ли натяга до цели.
Тесты: слабый натяг не стреляет; при равном натяге на большом экране дальности не хватает,
на маленьком хватает; цель выбирается по наименьшему углу и только в допуске.

`RoomEngine`: натяг колоды пальцем вниз (ограниченно, с резинкой); на отпускании три
исхода — выстрел в цель (полёт дугой → `onDealCard`), недолёт (карта тормозит, «не
долетело», возврат), промах по углу (край экрана, «Мииииимоо», возврат).
`dragMode.ts` получает режим `"sling"` для центра в режиме раздачи.
Драг одной карты из открытого веера на место игрока → тот же `onDealCard`.

### Этап 5. Кнопки, роли, доп. действия

`game/barActions.ts` — контекст расширяется (`dealMode`, `amIDealer`, `autoDealing`,
`myFanOpen`):
- игрок: main «Готов»/«Не готов», secondary «Ждите…» (disabled);
- дилер: main «Перемешать», secondary «Автораздача» ↔ «STOP».
Меню (гамбургер): дилеру — «Собрать все карты»; любому при раскрытом веере руки —
«Сортировать по масти» и «Сортировать по номиналу». «Раздать» из меню убирается.
Тесты на таблицу решений: роль × режим × автораздача × раскрытый веер.

Новый чистый `client/src/game/sortHand.ts`: `sortBySuit(cards)`, `sortByRank(cards)` на
`parseCard`. Тесты: состав не меняется, порядок мастей и номиналов стабилен, джокеры/
неизвестные ранги не роняют сортировку.

### Этап 6. Автораздача и сбор

Новый чистый `client/src/game/dealing.ts`:
- `dealOrder(seatIds, dealerId)` — от соседа по кругу, дилер последний;
- `autoDealPlan(order, cardsEach)` — два круга по одной карте;
- `collectOrder(seatIds, dealerId)` — по часовой от дилера (для анимации сбора);
- `AUTO_DEAL_INTERVAL_MS = 2000 / 3`.
Тесты: дилер последний, длина плана = игроки × 2, сбор идёт по часовой от дилера.

Драйвер автораздачи в `RoomScreen`: таймер шлёт `deal_card` для верхней карты текущему
получателю, останавливается по «STOP», по концу плана и на пустой колоде. Порядок карт
сохраняется, потому что берётся всегда верхняя и ждётся эхо состояния.

Анимация сбора: `collect_hands` + fx-событие; у всех карты летят с мест в колоду со
сдвигом по времени в порядке `collectOrder`.

## Проверка

- `cd server && npx vitest run` — включая новые тесты на `deal_card`, `set_hand_order`,
  `set_deck_fanned` и запрет `move_deck` в режиме раздачи.
- `cd client && npx tsc --noEmit && npx vitest run && npx vite build`.
- Живьём на `192.168.1.66:5173` (комната с ботами):
  1. дилер видит «Перемешать» / «Автораздача», игрок — «Готов» / «Ждите…»;
  2. двойной тап по колоде раскрывает веер на столе, у второго клиента тоже и рубашкой;
     стрелка сворачивает его в колоду; раскрытие руки сворачивает веер колоды;
  3. слабый натяг рогатки не стреляет; средний на большом экране не долетает; хороший
     попадает в бота — его счётчик растёт; выстрел мимо даёт «Мииииимоо»;
  4. «Автораздача» раздаёт по две карты, дилер последний, «STOP» прерывает; свои карты
     видно номиналом сразу;
  5. «Собрать все карты» — у обоих клиентов карты летят в колоду по часовой от дилера;
  6. сортировка руки по масти и по номиналу переживает перезаход в комнату.
