## 1. Что это

Мультиплеерная карточная игра для мобильных браузеров (2–32 игрока, колода 36/52).
Пиксельный кэжуал-стиль (Balatro-style), виртуальный стол. Стек: **Colyseus** (Node) +
**React/Vite** + **Pixi.js** (v8, императивный движок стола) + **Framer Motion** (меню/фон).
Свои аккаунты (recovery-код, без пароля), Firebase — задел на будущее, не используется.

**Игровых правил (GameEngine) ещё НЕТ.** Сделана вся обвязка: аккаунты, комнаты, дилер/
голосования, память комнат, и «физическая» работа с колодой (перемещение по зонам, веер,
растасовка, лица карт). Следующий большой шаг — сами правила игры.

## 2. Готовые фичи (этой серии работ)

### Стол и зоны (Pixi, `client/src/game/`)
- Стол — **весь экран** (овалы убраны). Канвас `absolute; inset:0`; топбар/баннер голосования/
  нижние кнопки — **оверлеи** поверх (см. `theme.css` `.table-screen`/`.room-canvas`/
  `.table-bottombar`). Размер стола статичен, пересчитывается только на реальный ресайз.
- Зоны (`layout.ts` → `RoundedRect`): **центр** (сверху-середина), **сейф-зона** (посередине-
  снизу), **рука** (у нижнего края, пока НЕдоступна для дропа). Три широкие (≥80% ширины)
  скруглённые полосы. Подсветка зон + подписи (ЦЕНТР/СЕЙФ-ЗОНА/РУКА) видны только при драге.
- Дроп-зоны (`dropZones.ts`): `center`/`safe` (droppable) и `hand` (недоступна). `pickDropZone`
  тестируется юнитами.

### Колода: перемещение
- Сервер хранит `GameState.deckLocation` = `"center"` или `sessionId` держателя сейф-зоны.
  Сообщение `move_deck {zone:"center"|"safe"}` — только дилер, только в лобби.
- Клиент: **драг-н-дроп** колоды дилером (Balatro-фил: приподнятие/инерция/крен). Дроп в
  недоступную зону (рука) → «ударный» отскок + текст «низяяя» по центру. Дабл-клик по колоде
  в центре → перенос в сейф-зону + авто-раскрытие веера. `deckZone` в клиенте (`deckZone.ts`):
  `center`/`safe`/`away` (чужая сейф-зона — колода скрыта).

### Колода: веер (`fan.ts` + движок)
- Дабл-клик в сейф-зоне раскрывает/собирает **веер-дугу**: наклон линейно до `maxAngleDeg`
  (30°), «почти арка». `fanCard(i,count,...)` — чистая математика, тестируется.
- **«Червячок»** для тесного веера (52 карты): когда шаг между картами мал (`fanCrowd()`),
  карты бегущей синус-волной сближаются/раздвигаются + дрожат. **Энергия** стартует высокой
  (`boost`) и спадает к базовой за `decayTime`=4с (быстро в начале, `energyEnvelope`).
  Полная анимация = волна+дрожь; умеренная = только волна и медленно. 36 карт (просторно) →
  эффект выключен.
- **Раскрытие под указателем**: тач (айфон) — **тык** (`pokeFan`, раскрывает ~5 карт на 2.5с,
  `pokeEnvelope`, + ре-энергия волны от точки тыка). Десктоп — **ховер мышью** (раскрытие
  следует за курсором, `hoverEnv` сглаживается). Тык на десктопе отключён (по `pointerType`).

### Колода: лица и растасовка
- **Лица карт** генерятся динамически (`card.ts` `parseCard`/`suitColor` + движок
  `makeCardFaceTexture`): ранг+масть по углам, крупный символ по центру, J/Q/K — буква-
  заглушка (картинки позже). Кэш текстур по `card|fourColor`. Рубашка (`backTex`) — отдельная
  сменяемая текстура.
- **Четырёхцветная колода** (♦ оранж, ♣ голубой) для слабовидящих: тумблер в меню, персист
  (`useFourColor`).
- **Кнопка «Перевернуть карты»** (`deckFaceUp`) — лицо/рубашка, независимо от веера/зоны,
  доступна дилеру пока есть колода (центр/сейф).
- **Настоящая растасовка**: спрайты привязаны к ИДЕНТИЧНОСТИ карты. Кнопка «Растасовать» шлёт
  `shuffle_deck`; сервер тасует массив, новый порядок приходит эхом → `setDeck` детектит
  реордер и анимирует **реальную перестановку** ПО ДЕЛЬТЕ каждой карты (`|new-old|`: близкие —
  коротко/незаметно, далёкие — долго высокой дугой), z-swap в апексе каждой карты. Пока летит
  запрос — **«сумбур»** (карты хаотично меняются местами, `startScramble`).

### Меню / настройки анимаций
- `useAnimationSettings`: `level` (full/moderate) + `speed` (1/2/3, только на full) + `shadows`.
  `resolveProfile` → `AnimationProfile` (движок). Тумблеры теней и четырёхцветной колоды в меню.
- Тени: ОДНА мягкая тень под всей колодой (не на карту — иначе накопление альфы). idle-анимация
  карт (лёгкое «дыхание»), гасится на умеренной.

### Комнаты, аккаунты, память
- Дилер (первый вошедший), голосования propose_dealer/propose_kick (вес дилера 1.5, таймаут).
- **Память комнаты** (см. `memory/project_room_memory.md`):
  - Комнаты НЕ диспоузятся сразу: `autoDispose=false` + таймер `EMPTY_ROOM_TTL_MS` (30 мин) —
    можно вернуться «в последнюю игру».
  - `lastRooms.ts` — per-account последняя комната (in-memory), `GET /accounts/:id/last-room`.
    Лобби показывает «↩ Вернуться в <код>».
  - **Дисконнект = ПАУЗА, не кик**: `onLeave` только ставит `connected=false`, игрок остаётся,
    дилерство сохраняется. `onJoin` при возврате тем же `accountId` усыновляет игрока (рука/
    дилерство переносятся на новый sessionId). Убраны allowReconnection/снапшоты/авто-передача.
  - Клиент: комната на саброуте `/room/<inviteCode>` (`roomRoute.ts`), персист «активной
    комнаты» в localStorage, **бесшовный авто-реконнект** при загрузке/обрыве (лоадер
    «Возвращаемся…»), обрыв (`room.onLeave/onError`) роняет room но авто-переподключается.
    Явный «Покинуть комнату» чистит персист.

## 3. Архитектура клиента (важное)

### `client/src/game/RoomEngine.ts` — императивный Pixi-движок (сердце стола, ~900 строк)
- Владеет ОДНИМ Pixi `Application`, тикером, всеми объектами. **Никакого @pixi/react** (прошлый
  подход крашился — см. `memory/project_scene_removed.md`). Карты — простые `CardVisual`
  `{body: CardBody, sprite, card, phase}`, мутируем сами.
- **Слои** (world.sortableChildren): стол(0) → подсветка зон(1) → тени(2) → карты(3,
  sortableChildren) → невидимый хит колоды(10000) → «низяяя»(5000).
- **Рендер-цикл спит в простое** (`wake`/`sleep`): рисуем только когда что-то движется. Условие
  сна учитывает shuffleAnim/scrambleAnim/press/reject/idle/fanWiggling. Любая новая непрерывная
  анимация ДОЛЖНА добавляться в это условие, иначе либо не играет, либо не гасит CPU.
- **`CardBody`** (`CardBody.ts`) — пружинная физика на каждый канал (x/y/rot/scale). Анимации
  двигают ЦЕЛИ (`setTarget`), пружины сглаживают. `snapTo` — телепорт.
- Публичные сеттеры (зовёт `RoomCanvas` из React через `useEffect`): `setDeck(string[])`,
  `setDeckZone`, `setDeckDraggable`, `setFourColor`, `setDeckFaceUp`, `startScramble`,
  `setOnDeckDoubleClick/Drop/DragChange`, `setAnimationProfile`, `resize`, `mount`, `destroy`.
- Указательные события (drag/tap/hover) — на невидимом `deckHit` (Container с hitArea) и на
  `app.stage` (move/up ловим на всей сцене). Различаем мышь/тач по `e.pointerType`.

### React-слой
- `RoomCanvas.tsx` — тонкий хост: монтирует движок 1 раз, прокидывает пропсы в него через
  эффекты. `deck` передаётся как массив; эффект завязан на `deckKey = deck.join(",")` (массив
  пересоздаётся каждый sync). `shuffleSignal` (счётчик) → `startScramble`.
- `RoomScreen.tsx` — состояние из `room.state` (sync на onStateChange), нижние кнопки, баннер
  голосования, проброс в RoomCanvas. Кнопки: `showCenterActions` (Готов/Раздать, только центр),
  `showDeckPlaceholder` (Вернуть в центр), `showDeckTools` (Растасовать/Перевернуть — пока есть
  колода).
- `App.tsx` — роутинг/авто-реконнект (`roomRoute.ts`), auth, меню. `AppMenu.tsx` — меню.

### Конфиг анимаций: `client/src/game/anim/config.ts`
Единый источник тайминга/физики. Ключевое для веера — `anim.fan.wiggle`:
`gap`/`ramp` (порог тесноты), `amp`/`cycles`/`freq`/`moderateFreq` (волна), `jitterRotAmp`/
`jitterFreq` (дрожь), `boost`/`decayTime` (энергия), `poke:{cards,amp,in,hold,out}` (раскрытие).
`anim.fan.maxAngleDeg`/`widthFactor` — форма веера. Все числа «на глаз» — тюнятся тут.

**Мёртвый код:** `client/src/game/choreo/` (ShuffleChoreography/SpinChoreography) и поля
профиля `shuffleVariant/stagger/jitter/scaleBump` — остались от старой косметической растасовки,
движком больше НЕ используются (тесты choreo ещё есть). Можно вычистить отдельным коммитом.

## 4. Архитектура сервера (`server/src/`)

- `CardRoom.ts` — комната Colyseus. Env-тюнинг (лениво из `process.env`, для тестов):
  `VOTE_TIMEOUT_MS`, `EMPTY_ROOM_TTL_MS`. `autoDispose=false` + `disposeTimer`.
- `GameState.ts` — схемы (`@colyseus/schema`). Player: id(=accountId)/name/isDealer/isReady/
  connected/hand. GameState: players(MapSchema по sessionId)/deckType/phase/inviteCode/isPublic/
  activeProposal/deck(ArraySchema)/deckLocation.
- `accounts.ts` — **персист на диск** (`server/data/accounts.json`, debounced save; в .gitignore).
  Единственная дисковая персистентность.
- `lastRooms.ts`, `inviteCodes.ts`, `publicRooms.ts` — in-memory реестры (module-level Map).
- `index.ts` — express + HTTP-эндпоинты (accounts CRUD, `/rooms/by-code/:code`, `/rooms/public`,
  `/accounts/:id/last-room`, `/health`). CORS `*`.
- Тесты: `*.test.ts` рядом. `CardRoom.test.ts` мокает `./accounts.js` (чтобы accountId в onAuth
  распознавался как валидный) и ставит короткие env-таймауты.

**Важное ограничение:** вся память комнат/реестры — в ОЗУ. Рестарт сервера стирает комнаты
(и last-room). Диск-персистентность комнат — осознанно НЕ делали (большая задача).

## 6. Тонкости, на которые легко наступить

- **Спрайты карт привязаны к идентичности** (`CardVisual.card`), не к индексу. `setDeck` делает
  `reconcileByIdentity` (переиспользует спрайты по карте) — это то, что делает растасовку
  «настоящей». Не ломай это при добавлении раздачи/руки.
- **`deckLocation` хранит sessionId** — при паузе/возврате перецепляется на новый sessionId
  (`onJoin`). Любая новая sessionId-привязанная стейт-переменная должна так же переживать возврат.
- **Оверлеи чистые от layout**: не клади в поток ничего, что меняет высоту и заставит канвас
  ресайзиться (это дёргало стол — фикс `fc8f0d3`/`77f762c`).
- **Клиент уже получает весь `deck` (порядок)** от сервера (schema синкается всем) — это не новая
  утечка, но при добавлении «скрытых» карт учитывай, что клиент видит порядок колоды.
